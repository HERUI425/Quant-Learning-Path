### Phase 1: Robust Cross-Sectional Data Pipeline

开发了一个生产级的数据提取管道，基于 **Tushare API** 实现高效、稳定的金融数据获取。

**核心特性：**

- **反爬虫防御机制**：采用基于 `trade_cal` 日历的 **Calendar-Stepping Stepper**，系统性地规避高频请求封禁和速率限制。
- **张量对齐（Tensor Alignment）**：利用 Pandas `pivot_table` 的向量化操作，将非对称的交易日志转换为同步的、身份映射的二维截面矩阵  
  $\mathbf{X} \in \mathbb{R}^{T \times N}$，从根本上避免了前视偏差（look-ahead bias，如 `.shift(-1)`）。

**技术亮点**：
- 完全向量化处理，性能优异

## 1. 核心因子构建与数据预处理

本模块基于原始行情数据（收盘价、成交量、成交额），通过滚动窗口与截面变换，构建了两个核心 Alpha 因子，并完成了初步的异常值清洗。

### 1.1 收益率矩阵计算
在因子构建前，首先将绝对价格矩阵变换为相对收益率矩阵，以消除资产绝对价格标签的影响。
$$R_{t} = \frac{P_{t} - P_{t-1}}{P_{t-1}}$$

* **实现说明**：使用 `pct_change()` 函数计算各资产收盘价的百分比变化率 $df\_return$。


### 1.2 Factor 1: 5日滚动量价相关性因子 (Price-Volume Divergence, PVD)
* **设计逻辑**：该因子旨在捕捉量价配合关系的短期异动。经典的量价理论认为，价格上涨伴随成交量放大（量价齐升）是趋势健康的信号，而量价背离则可能预示着趋势衰竭。
* **数学表达**：对每个资产，在时间戳 $t$ 处，取过去 $N=5$ 天的收盘价序列 $\mathbf{P} = \{P_{t-4}, \dots, P_t\}$ 与成交量序列 $\mathbf{V} = \{V_{t-4}, \dots, V_t\}$，计算其**皮尔逊线性相关系数（Pearson Correlation）**：
PVD_t = [ Σ(i=0→4) (P_{t-i} - P̄)(V_{t-i} - V̄) ] / [ √( Σ(i=0→4) (P_{t-i} - P̄)² ) × √( Σ(i=0→4) (V_{t-i} - V̄)² ) ]

**设计备注**：本系统选用皮尔逊相关系数而非斯皮尔曼（Spearman）秩相关系数。由于系统采用 5 日短周期滚动窗口（N=5），样本量较小，斯皮尔曼的秩排序容易因离散化导致因子值频繁跳跃；同时，皮尔逊相关系数能更敏感地捕捉到行情暴涨/暴跌时“放量”的绝对幅度，更符合 Price Action 对极端量价结构的定义。


### 1.3 Factor 2: 非流动性因子 (Amihud Illiquidity Proxy)
* **设计逻辑**：借鉴经典的 Amihud (2002) 非流动性指标。该因子衡量的是**单位成交额引起的绝对收益率变化**（即每单位资金交易引起的“价格冲击”）。因子值越大，说明市场深度越越弱，流动性越差。
* **数学表达**：
$$\text{Illiquidity}_t = \frac{|R_t|}{\text{Amount}_t / 1,000,000}$$

* **健壮性处理（防爆机制）**：
  * **缩放处理**：由于原始成交额（Amount）绝对数值过大，直接作分母会导致因子值缩减至 $10^{-7}$ 级别的科学计数法，影响后续矩阵计算的数值稳定性。因此将成交额统一除以 $1,000,000$（以百万为单位）。
  * **零值替换**：在市场停牌或无成交量的极端情况下，$\text{Amount} = 0$ 会导致程序触发 `ZeroDivisionError`。代码中使用 `.replace(0, np.nan)` 将分母为 0 的异常值强制转换为 `NaN`（空值），确保后续截面计算时该资产被自动剔除，保障系统稳定性。
  * ## 2. 因子数据预处理 (Data Preprocessing)

原始计算出的因子矩阵往往包含大量的噪点、极端异常值以及缺失值。为了保证后续线性回归或机器学习模型的稳定性，必须在横截面上对因子进行清洗。

### 2.1 MAD 绝对中位数去极值 (Median Absolute Deviation)
* **为什么要用 MAD 去极值，而不是均值方差法（3-Sigma）？**
  * **传统 3-Sigma 的致命缺陷**：3-Sigma 依赖均值和方差。然而，金融数据（尤其是加密货币或个股）的原始因子往往存在极端的尖峰厚尾特征。几个巨大的异常值就会把“均值”拉偏，把“方差”撑大，从而导致原本合理的阈值失效，无法有效识别真正的异常值。
  * **MAD 的抗噪优势（鲁棒性）**：MAD 全程采用**中位数**来衡量因子的离散程度。由于中位数对极值极不敏感（哪怕市场上有几个资产的因子值翻了 1000 倍，中位数也几乎不动），因此它能提供一个极其稳定、不受极端行情污染的基准线。
* **工程处理逻辑**：
  1. 找出截面上所有资产因子的中位数 $M$。
  2. 计算每个因子值偏离中位数的绝对距离，再取这些距离的中位数，得到 $\text{MAD}$。
  3. 将偏离 $M$ 超过 $n$ 倍 $\text{MAD}$（通常取 $n=3$ 或 $5$）的极端值，强制拉回（Winsorize）到边界值上。


### 2.2 中位数缺失值填充 (Median Imputation)
* **为什么要用中位数填充，而不是直接删掉或者填 0？**
  * **不能直接剔除**：如果因为某个资产在当天缺少某些数据（例如新币上市、停牌、或者流动性极低导致因子算出来是 `NaN`）就直接删掉这一行，会导致多因子横截面矩阵的维度不一致，破坏回测系统的整体连续性。
  * **不能盲目填 0**：如果直接填 0，在数学上等同于默认该资产的因子表现处于“绝对中性”或“无特征”。但在某些因子（如非流动性因子）中，0 往往具有特定的业务含义（比如交易量极大），盲目填 0 会引入严重的人为噪声。
  * **选择中位数的考量**：填入当天全市场所有资产的中位数，是目前统计学上最稳妥的“免责声明”。它既保留了该资产在当天的交易资格，又确保了该资产不会因为缺失值而在多空组合排名中产生任何极端的偏向，最大程度维持了因子的横截面分布形态。
