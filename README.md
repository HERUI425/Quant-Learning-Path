## 1. 核心因子构建与数据预处理

本模块基于原始行情数据（收盘价、成交量、成交额），通过滚动窗口与截面变换，构建了两个核心 Alpha 因子，并完成了初步的异常值清洗。

### 1.1 收益率矩阵计算
在因子构建前，首先将绝对价格矩阵变换为相对收益率矩阵，以消除资产绝对价格标签的影响。
$$R_{t} = \frac{P_{t} - P_{t-1}}{P_{t-1}}$$

* **实现说明**：使用 `pct_change()` 函数计算各资产收盘价的百分比变化率 $df\_return$。


### 1.2 Factor 1: 5日滚动量价相关性因子 (Price-Volume Divergence, PVD)
* **设计逻辑**：该因子旨在捕捉量价配合关系的短期异动。经典的量价理论认为，价格上涨伴随成交量放大（量价齐升）是趋势健康的信号，而量价背离则可能预示着趋势衰竭。
* **数学表达**：对每个资产，在时间戳 $t$ 处，取过去 $N=5$ 天的收盘价序列 $\mathbf{P} = \{P_{t-4}, \dots, P_t\}$ 与成交量序列 $\mathbf{V} = \{V_{t-4}, \dots, V_t\}$，计算其**皮尔逊线性相关系数（Pearson Correlation）**：
PVD_t = [ Σ(i=0→4) (P_{t-i} - P̄)(V_{t-i} - V̄) ] / [ √( Σ(i=0→4) (P_{t-i} - P̄)² ) × √( Σ(i=0→4) (V_{t-i} - V̄)² ) ]

**设计备注**：本系统选用皮尔逊相关系数而非斯皮尔曼（Spearman）秩相关系数。由于系统采用 5 日短周期滚动窗口（N=5），样本量较小，斯皮尔曼的秩排序容易因离散化导致因子值频繁跳跃；同时，皮尔逊相关系数能更敏感地捕捉到行情暴涨/暴跌时“放量”的绝对幅度，更符合 Price Action 对极端量价结构的定义。


### 1.3 Factor 2: 非流动性因子 (Amihud Illiquidity Proxy)
* **设计逻辑**：借鉴经典的 Amihud (2002) 非流动性指标。该因子衡量的是**单位成交额引起的绝对收益率变化**（即每单位资金交易引起的“价格冲击”）。因子值越大，说明市场深度越越弱，流动性越差。
* **数学表达**：
$$\text{Illiquidity}_t = \frac{|R_t|}{\text{Amount}_t / 1,000,000}$$

* **健壮性处理（防爆机制）**：
  * **缩放处理**：由于原始成交额（Amount）绝对数值过大，直接作分母会导致因子值缩减至 $10^{-7}$ 级别的科学计数法，影响后续矩阵计算的数值稳定性。因此将成交额统一除以 $1,000,000$（以百万为单位）。
  * **零值替换**：在市场停牌或无成交量的极端情况下，$\text{Amount} = 0$ 会导致程序触发 `ZeroDivisionError`。代码中使用 `.replace(0, np.nan)` 将分母为 0 的异常值强制转换为 `NaN`（空值），确保后续截面计算时该资产被自动剔除，保障系统稳定性。
  * ## 2. 因子数据预处理 (Data Preprocessing)

原始计算出的因子矩阵往往包含大量的噪点、极端异常值以及缺失值。为了保证后续线性回归或机器学习模型的稳定性，必须在横截面上对因子进行清洗。

### 2.1 MAD 绝对中位数去极值 (Median Absolute Deviation)
* **为什么要用 MAD 去极值，而不是均值方差法（3-Sigma）？**
  * **传统 3-Sigma 的致命缺陷**：3-Sigma 依赖均值和方差。然而，金融数据（尤其是加密货币或个股）的原始因子往往存在极端的尖峰厚尾特征。几个巨大的异常值就会把“均值”拉偏，把“方差”撑大，从而导致原本合理的阈值失效，无法有效识别真正的异常值。
  * **MAD 的抗噪优势（鲁棒性）**：MAD 全程采用**中位数**来衡量因子的离散程度。由于中位数对极值极不敏感（哪怕市场上有几个资产的因子值翻了 1000 倍，中位数也几乎不动），因此它能提供一个极其稳定、不受极端行情污染的基准线。
* **工程处理逻辑**：
  1. 找出截面上所有资产因子的中位数 $M$。
  2. 计算每个因子值偏离中位数的绝对距离，再取这些距离的中位数，得到 $\text{MAD}$。
  3. 将偏离 $M$ 超过 $n$ 倍 $\text{MAD}$（通常取 $n=3$ 或 $5$）的极端值，强制拉回（Winsorize）到边界值上。


### 2.2 中位数缺失值填充 (Median Imputation)
* **为什么要用中位数填充，而不是直接删掉或者填 0？**
  * **不能直接剔除**：如果因为某个资产在当天缺少某些数据（例如新币上市、停牌、或者流动性极低导致因子算出来是 `NaN`）就直接删掉这一行，会导致多因子横截面矩阵的维度不一致，破坏回测系统的整体连续性。
  * **不能盲目填 0**：如果直接填 0，在数学上等同于默认该资产的因子表现处于“绝对中性”或“无特征”。但在某些因子（如非流动性因子）中，0 往往具有特定的业务含义（比如交易量极大），盲目填 0 会引入严重的人为噪声。
  * **选择中位数的考量**：填入当天全市场所有资产的中位数，是目前统计学上最稳妥的“免责声明”。它既保留了该资产在当天的交易资格，又确保了该资产不会因为缺失值而在多空组合排名中产生任何极端的偏向，最大程度维持了因子的横截面分布形态。
  * ## 3. 因子横截面市值中性化 (Market Cap Neutralization)

在量化投资中，许多阿尔法因子（如非流动性因子 Illiquidity）天生与资产的“市值大小”存在强烈的线性相关性。小市值公司由于流动性较差，往往天然具有较高的非流动性指标。如果不剔除这种伴生的“市值效应”，我们的策略最终可能只是简单地在做多小盘股、做空大盘股，从而承担了巨大的小市值风险暴露（Size Risk Exposure）。

为了挖掘出真正纯净的选股信号，本系统在每天的横截面上，对因子执行了基于 OLS 线性回归的市值中性化处理。

### 3.1 核心步骤与工程落地

#### 步骤一：构建本地对数市值矩阵
* **处理逻辑**：首先将所有股票的历史市值数据（Total Market Value）拼接并进行数据清洗（剥离交易所后缀，统一时间戳格式）。
* **对数变换**：在提取出市值矩阵后，对其取自然对数：
    ln_Size = ln(Market_Value)
* **为什么要取对数（ln）？**
    金融市场的市值分布极度不均匀，大市值和小市值之间往往相差几个数量级。取对数可以使市值数据更接近正态分布，降低极端大盘股对回归线的拉拽作用，从而让线性回归模型更加稳健。

#### 步骤二：逐日横截面 OLS 线性回归（残差提取）
* **回归模型设计**：
    在每一个交易日（t），在全市场横截面上，以原始非流动性因子值（Y）作为因变量，以当天对数市值（X）作为自变量，建立带有截距项（Alpha）的普通最小二乘法（OLS）回归方程：
    
    Y = Alpha + Beta * ln_Size + Residual (残差)

* **为什么要强行加上截距项（sm.add_constant）？**
    如果不加截距项，回归线会被强行拖到坐标轴原点（0,0），这会导致拟合出的 Beta 严重失真，提取出的残差中依然会残留市值成分。

#### 步骤三：提取“纯净”因子值
* **残差即因子**：回归后得到的“残差（Residual）”，在数学上代表了原始非流动性因子中**无法被市值大小解释的部分**。
* **最终效果**：我们用这个残差作为清洗后全新的“中性化非流动性因子（df_illiq_neutral）”。此时，该因子的横截面均值被归零，且与市值因子的相关性绝对为 0，彻底切断了市值效应的干扰。

### 3.2 代码工程中的健壮性设计（防爆机制）

1.  **动态样本量检查**：
    代码中设置了 `if valid_idx.sum() < 3: continue` 的边界判断。由于回归方程包含两个未知数（Alpha 和 Beta），数学上至少需要 3 个样本点才能进行有效的线性拟合。如果当天因为停牌等原因导致有效股票样本太少，则直接跳过该日，防止矩阵计算报错引发程序崩溃。
2.  **规避 Pandas 隐式转换警告**：
    显示开启了 `pd.set_option('future.no_silent_downcasting', True)`。在向空 DataFrame 逐行填充残差对象时，Pandas 内部会发生数据类型的隐式转换（Object 转换为 Float64）。显式开启此项可以规避未来 Pandas 版本的兼容性告警，确保生产环境日志的干净整洁。

### Phase 3：Factor Validity Test
## 1. 因子有效性检验实证分析 (Factor Validity Empirical Analysis)

本模块对清洗、中性化后的核心因子进行了全样本期（共 104 个交易日）的逐日横截面 Rank IC 检验。通过对齐因子矩阵与未来一期收益率矩阵（Target Y），系统排除了周末及停牌干扰，得到了各因子的真实选股能力指标。

### 1.1 核心检验指标对比实测

经过 104 个交易日的横截面滚动测算，非流动性因子（Illiquidity）与 5日滚动量价相关性因子（PVD）的实测核心数据对比如下：

| 统计指标 | Factor 1: PVD (量价相关性) | Factor 2: Illiquidity (非流动性) | 量化业务含义对比解析 |
| :--- | :--- | :--- | :--- |
| **Mean Rank IC** | **-2.770%** | **-8.066%** | **Illiquidity 选股能力远强于 PVD**。Illiquidity 绝对值高出 5% 的优秀门槛；PVD 负向预测能力较弱。 |
| **Std IC** | **32.526%** | **32.550%** | 两者在时间序列上的波动率几乎一致（32.5% 左右），说明金融市场短期噪声极大，单因子表现均存在宏观周期切换的尾部风险。 |

### 1.2 💡 因子缺陷诊断与 Version 2.0 明确迭代逻辑

结合两个因子的实测表现，我们直接找到了当前 Version 1.0 系统的痛点，并据此制定了下一阶段明确的迭代与加因子逻辑：

#### 1. 为什么 PVD (5日量价相关性) 会在全样本期内“失效”？
* **短周期窗口（5日）的局限**：5日滚动窗口（`rolling(5)`）虽然敏锐度高，但在 104 个交易日的跨度内，市场大部分时间处于 Brooks 价格行为学定义的**“交易区间（Trading Range）”**（震荡市）。在震荡市中，量价配合经常失效，假突破频繁，导致 5日 相关性指标产生大量无序的随机噪声，拉低了 Mean Rank IC。
* **线性相关性的局限**：量价关系往往是非线性的（例如：极度缩量到一定地步才会引发变盘），皮尔逊相关系数难以捕捉这种非线性结构，导致 T 统计量不显著（-0.8475）。

#### 2. 为什么 Illiquidity 显著，但整体 IR 指标偏低？
* **环境不解耦**：Illiquidity 因子表现出极显著的负相关特性（选股抱团流动性好的资产）。但在牛市亢奋期（全盘普涨、资金溢出）和流动性危机期（资金极度抱团大盘蓝筹），因子的表现截然相反。两股力量在 104 天内反复拉锯，导致 Std IC 被撑大到了 32.55%，压低了 IR。

#### 3. 下一步模型明确迭代路线（加因子与合成方案）
为了打破当前单因子作战、噪声过高的困境，后续迭代锁定了以下两个方向：

* **明确加因子方向：引入市场状态/动量开关因子**
    * *逻辑*：我们需要明确加入能区分“趋势（Trend）”和“交易区间（Trading Range）”的动量或波动率类因子（例如基于 ATR 振幅或通道宽度的因子）。
    * *作用*：用新因子作为“环境路由器”。当新因子识别出市场处于强烈趋势时，激活并加大 PVD 因子权重（捕捉放量突破）；当识别出市场处于震荡区间时，屏蔽 PVD 因子，转向依赖 Illiquidity 因子，从而强行压低整体因子的时间序列标准差。
* **多因子合成策略升级**
    * *逻辑*：不再依赖单因子。下一步将 Illiquidity 与 PVD 统一乘以 -1 进行正向化，通过 **IC 序列风险加权（IR 加权）** 或回归残差合成的方式，将两个因子融合成一个综合 Alpha 因子，利用因子间的低相关性对冲掉单一量价因子的随机噪声，目标将综合 IR 提升至 0.5 以上。
